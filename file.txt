connection.php: Questo file gestisce la connessione al database. Viene utilizzato da altri file nel progetto per stabilire una connessione al database e interagire con esso

index.php: Questo è il file principale del progetto. Viene utilizzato come entry point per l'applicazione web. Contiene il layout generale dell'applicazione.

createUsersDatabase.sql: Questo file è uno script SQL utilizzato per creare il database con tutte le relazioni e informazioni pertinenti necessarie ai fini dell'applicazione.

curious.php: In tale file mostriamo all'interno di un iframe un video scelto randomicamente dalla relazione Videos del database. Quest'ultima è stata popolata con video appartenenti ad una playlist pubblica di YouTube creata per l'occasione.

homepage.php: Rappresenta la prima pagina dell'applicazione ad essere visualizzata dall'ipotetico user. Da qui è possibile accedere tramite Login o registrarsi con Signup.



css  (directory)

login.css: Questo file contiene le regole di stile (CSS) per la pagina di login e di signup. Definisce l'aspetto visivo degli elementi HTML sulla pagina di accesso, come campi di input, pulsanti e layout generale. Il file viene incluso nella pagina di accesso (login.php) per fornire un design appetibile.
index.css: Questo file contiene le regole di stile (CSS) specifiche per la pagina di indice (index.php) del progetto.

js  (directory)

registration.js: Questo file contiene il codice JavaScript per la registrazione degli utenti. Può gestire la validazione dei campi di input, l'invio dei dati al server (in modalità asincrona mediante l'utilizzo della tecnologia AJAX) per la registrazione e altre funzioni correlate alla registrazione degli utenti. Viene incluso nella pagina di registrazione (registration.php) per fornire funzionalità interattive e di controllo dei dati inseriti dagli utenti.
index.js: Contiene il codice JavaScript specifico per "index.php". Include funzioni e logica per manipolare gli elementi HTML, gestire gli eventi dell'utente, effettuare richieste AJAX o altre operazioni dinamiche sulla pagina.
login.js: Contiene il codice JavaScript specifico per "login.php". Include funzioni per la validazione dei campi di accesso, il controllo dell'input dell'utente, la gestione degli eventi di login e altre operazioni correlate all'autenticazione degli utenti.
requestHandler.js: Contiene il codice JavaScript relativo alla dinamicità lato frontend della gestione delle richieste d'amicizia implementate nel Social.

pages  (directory)

printRequest.php: Questo file è responsabile per la visualizzazione delle richieste di amicizia. Contiene il codice per recuperare le richieste di amicizia dal database e mostrare i dati corrispondenti nella pagina. È incluso in una parte del sito web in cui gli utenti possono accedere alle richieste di amicizia che hanno ricevuto e decidere se accettarle o rifiutarle.
addFriendPopup.php: Gestisce la visualizzazione di una finestra popup per aggiungere un amico. All'apertura di questa è possibile inviare la richiesta a persone registrate nel Social con cui non si ha ancora un rapporto di amicizia e nemmeno una richiesta pendente (caso gestito con la sostituzione del bottone di aggiunta con quello raffigurante una clessidra)
login.php: Rappresenta la pagina di accesso dell'applicazione. Include un modulo di accesso per l'utente, campi per l'inserimento delle credenziali, pulsanti per il login e funzionalità di registrazione.
friendRequests.php: Gestisce lato server le informazioni relative alle richieste di amicizia pendenti.
main.php: Rappresenta l'elemento HTML cuore dell'applicazione.
leftContainer.php: Rappresenta la sezione sinistra della pagina principale.
addMessage.php: Gestisce l'invio di un nuovo messaggio. Include il codice per ricevere i dati del messaggio inviato tramite un modulo HTML, effettua la validazione dei dati in modalità asincrona e delega il salvataggio del messaggio nel DB al file backAddMessage.php.
printPost.php: Gestisce la visualizzazione di un singolo post. Include il codice per recuperare le informazioni del post dal database, come il testo del post, l'autore, il timestamp e altri dettagli. Il file può formatta tali informazioni all'interno di un layout specifico e mostra il post nella pagina desiderata.
notificationPopup.php: Gestisce la visualizzazione di una finestra popup per le notifiche non lette. Una volta visualizzate verranno eliminate dal DB e dunque anche dal relativo popup lato client.
printMessage.php: Contiene la funzione per la visualizzazione di un messaggio specifico.
printNotification.php: Gestisce la visualizzazione di una notifica specifica. Potrebbe includere il codice per recuperare le informazioni sulla notifica dal database, come il tipo di notifica, l'autore, il timestamp e altri dettagli pertinenti. Il file può formattare queste informazioni e mostrare la notifica nella pagina desiderata, ad esempio in un'area dedicata alle notifiche degli utenti.
messagePopup.php: Gestisce la visualizzazione di una finestra popup per inviare un messaggio. Può contenere il codice HTML e PHP per mostrare un modulo di inserimento dei dati del destinatario, l'oggetto del messaggio e il corpo del messaggio. Una volta compilato il modulo, il file potrebbe contenere anche il codice per elaborare le informazioni inviate e inviare il messaggio corrispondente al destinatario.
preparePrintPost.php: Prepara i dati necessari per la visualizzazione di un post. Potrebbe contenere il codice per recuperare informazioni aggiuntive sul post, come i commenti, il numero di like o altre informazioni correlate. Il file prepara questi dati in un formato adatto per la visualizzazione del post e li rende disponibili per l'inclusione nel file printPost.php o in altri file correlati.
rightContainer.php: Rappresenta il contenuto o la sezione destra di una pagina specifica. Può contenere elementi come suggerimenti, elenchi di amici, annunci o altre informazioni pertinenti alla pagina. Viene incluso nelle pagine in cui è necessario visualizzare il contenuto nella sezione destra, ad esempio nella pagina principale o in una pagina del profilo utente.
middleContainer.php: Rappresenta il contenuto o la sezione centrale di una pagina specifica. Può contenere il contenuto principale della pagina, come un feed di post, una lista di messaggi o altre informazioni pertinenti. Viene incluso nelle pagine in cui è necessario visualizzare il contenuto nella sezione centrale, ad esempio nella pagina principale o in una pagina del profilo utente.
printChat.php: Gestisce la visualizzazione di una chat specifica. Potrebbe includere il codice per recuperare i messaggi di chat dal database, organizzarli in ordine cronologico e formattarli per la visualizzazione all'interno di una chat. Può anche includere funzionalità come l'aggiornamento in tempo reale della chat tramite l'uso di tecnologie come WebSocket.
addQuote.php: Gestisce l'aggiunta di una citazione al sistema. Può includere il codice per ricevere i dati della citazione, come l'autore e il testo, e salvarli nel database per essere visualizzati successivamente.
createPost.php: Gestisce la creazione di un nuovo post nel sistema. Può includere il codice per ricevere i dati del post, come il testo, le immagini o altri contenuti, effettuare la validazione dei dati e salvare il post nel database per essere visualizzato successivamente.
buttonPrintPostAndPopup.php: Gestisce la visualizzazione di un pulsante per stampare un post e la visualizzazione di una finestra popup associata. Può includere il codice per creare il pulsante e la finestra popup, nonché per gestire le interazioni dell'utente, come il clic sul pulsante per avviare la stampa o l'apertura della finestra popup per la visualizzazione del post.
quoteOfTheDay.php: Gestisce la visualizzazione di una citazione del giorno. Può includere il codice per recuperare una citazione dal database o da una fonte esterna, formattarla in modo appropriato e mostrarla nella pagina o in una sezione dedicata dell'applicazione.
setGlobalVariables.php: Contiene il codice per impostare variabili globali nell'applicazione. Può essere incluso all'inizio di altre pagine o script per rendere disponibili determinate variabili in tutto il progetto. Ad esempio, potrebbe essere utilizzato per impostare variabili globali come il percorso delle immagini, il percorso del file di connessione al database o altre variabili di configurazione che devono essere accessibili in tutto il codice dell'applicazione.
registration.php: Gestisce la registrazione di un nuovo utente nel sistema. Può includere il codice per ricevere i dati inseriti dall'utente tramite un modulo di registrazione, effettuare la validazione dei dati, creare un nuovo account utente nel database e reindirizzare l'utente a una pagina di conferma o di accesso successiva.
feeds.php: Gestisce la visualizzazione dei feed di contenuti nell'applicazione. Può includere il codice per recuperare i feed dal database o da altre fonti, organizzarli e mostrarli nella pagina principale o in un'area dedicata. I feed potrebbero essere post, aggiornamenti, notizie o altre informazioni rilevanti per gli utenti dell'applicazione.
navigationBar.php: Rappresenta la barra di navigazione presente nelle pagine del progetto. Può includere i link o le icone per le diverse sezioni o le pagine dell'applicazione, come il profilo utente, le impostazioni, la home page, le notifiche, i messaggi e altre funzionalità. Il file viene incluso nelle pagine per fornire una navigazione coerente e facilitare l'accesso alle diverse sezioni dell'applicazione.
messages.php: Gestisce la visualizzazione dei messaggi degli utenti. Può includere il codice per recuperare i messaggi dal database e formattarli per la visualizzazione all'interno di una chat o di una lista di messaggi. Può anche includere funzionalità come l'invio di nuovi messaggi o l'eliminazione dei messaggi esistenti.
sidebar.php: Rappresenta la barra laterale di navigazione presente nelle pagine del progetto. Può includere collegamenti a diverse sezioni del sito, come il profilo utente, la pagina principale, le impostazioni dell'account, le notifiche o altre funzionalità. Il file viene incluso nelle pagine del progetto per fornire una navigazione coerente e facilitare l'accesso alle diverse sezioni dell'applicazione.

script  (directory)
acceptRequest.php: Questo file gestisce l'accettazione di una richiesta di amicizia da parte di un utente. Può includere il codice per elaborare la richiesta e aggiornare lo stato dell'amicizia nel database, consentendo ai due utenti di diventare amici nella piattaforma.
backQuote.php: Questo file gestisce l'aggiunta di una citazione alla piattaforma. Può includere il codice per ricevere i dati della citazione, come l'autore e il testo, e salvarli nel database per essere visualizzati successivamente.
printPhoto.php: Questo file gestisce la visualizzazione di una foto specifica. Può includere il codice per recuperare le informazioni sulla foto dal database, come il percorso del file immagine, il nome dell'utente che ha caricato la foto e altri dettagli correlati. Il file può formattare queste informazioni per mostrare la foto nella pagina desiderata.
backAddMessage.php: Questo file gestisce l'aggiunta di un messaggio inviato da un utente. Può includere il codice per ricevere i dati del messaggio inviato, come il destinatario, il testo del messaggio e il mittente, e salvare il messaggio nel database per la visualizzazione successiva.
getUserLike.php: Questo file gestisce l'ottenimento delle informazioni sui "Mi piace" di un utente. Può includere il codice per recuperare le informazioni sui like associati a un utente specifico dal database e fornire i dati corrispondenti per essere utilizzati nelle pagine o nelle sezioni pertinenti.
logout.php: Questo file gestisce la disconnessione di un utente dalla piattaforma. Può includere il codice per terminare la sessione dell'utente, rimuovendo le informazioni di accesso salvate e reindirizzando l'utente alla pagina di accesso o a un'altra pagina specificata.
showProfiles.php: Questo file gestisce la visualizzazione dei profili utente. Può includere il codice per recuperare le informazioni sui profili utente dal database e presentare i dati corrispondenti nella pagina dedicata alla visualizzazione dei profili.
convertTimeFromTimestampToTime.php: Questo file contiene una funzione o un codice per convertire un timestamp (un valore numerico che rappresenta una data e un'ora specifiche) in un formato leggibile, come una stringa di data e ora. Viene utilizzato per formattare correttamente le informazioni temporali visualizzate nell'applicazione, ad esempio per mostrare la data e l'ora di un post o di un messaggio in un formato comprensibile agli utenti.
addComment.php: Questo file gestisce l'aggiunta di un commento a un post o a una foto. Può includere il codice per ricevere i dati del commento, come il testo del commento, l'ID del post o della foto e l'utente che ha inserito il commento, e salvare il commento nel database per la visualizzazione successiva.
getArrayPhotoLike.php: Questo file gestisce l'ottenimento di un array contenente le informazioni sui "Mi piace" ricevuti da una foto. Può includere il codice per recuperare le informazioni sui like associati a una foto specifica nel database e restituire un array di dati corrispondenti.
backRegistration.php: Questo file gestisce il processo di registrazione di un nuovo utente. Può includere il codice per ricevere i dati di registrazione, come l'username, l'email e la password, effettuare le opportune verifiche e salvare le informazioni utente nel database per consentire l'accesso
backAddFriend.php: Questo file gestisce l'aggiunta di un amico nel sistema. Può includere il codice per ricevere i dati dell'utente che si desidera aggiungere come amico, eseguire le verifiche necessarie (ad esempio, verificare se l'utente esiste nel database) e creare l'associazione di amicizia tra gli utenti nel database. Può anche gestire l'invio di notifiche o aggiornamenti correlati all'aggiunta dell'amico.
leaveLike.php: Questo file gestisce l'azione di mettere "Mi piace" a una foto o a un post. Può includere il codice per ricevere l'ID dell'oggetto (foto o post) a cui l'utente desidera lasciare il like, effettuare i controlli necessari e salvare l'informazione nel database, indicando che l'utente ha messo "Mi piace" all'oggetto specificato.
backLogin.php: Questo file gestisce il processo di accesso degli utenti al sistema. Può includere il codice per ricevere le credenziali di accesso, come l'username e la password, e verificare se corrispondono ai dati salvati nel database. Può anche gestire la creazione della sessione per l'utente autenticato e reindirizzarlo alla pagina principale o a un'altra pagina specificata.
discardRequest.php: Questo file gestisce il rifiuto di una richiesta di amicizia da parte di un utente. Può includere il codice per elaborare la richiesta e rimuoverla dal sistema, aggiornando lo stato della richiesta nel database.
removeLike.php: Questo file gestisce la rimozione di un "Mi piace" da una foto o un post. Può includere il codice per identificare il like specifico da rimuovere, aggiornare le informazioni corrispondenti nel database e gestire eventuali operazioni aggiuntive correlate alla rimozione del like.
showComments.php: Questo file gestisce la visualizzazione dei commenti associati a un post o a una foto. Può includere il codice per recuperare i commenti dal database, organizzarli in ordine cronologico e formattarli per la visualizzazione nella pagina corrispondente.
clearNotifications.php: Questo file gestisce la cancellazione delle notifiche degli utenti. Può includere il codice per identificare le notifiche specifiche da cancellare, ad esempio in base all'utente che le ha ricevute o al tipo di notifica, e rimuoverle dal database. Questo file può essere chiamato quando un utente decide di cancellare tutte le notifiche o quando viene visualizzata una determinata pagina o azione che richiede la rimozione delle notifiche.
setGlobalVariables.php: Questo file contiene il codice per impostare variabili globali nell'applicazione. Può essere incluso all'inizio di altre pagine o script per rendere disponibili determinate variabili in tutto il progetto. Ad esempio, potrebbe essere utilizzato per impostare variabili globali come il percorso delle immagini, il percorso del file di connessione al database o altre variabili di configurazione che devono essere accessibili in tutto il codice dell'applicazione.
backPost.php: Questo file gestisce l'elaborazione delle informazioni inviate tramite un modulo per creare un nuovo post. Può contenere il codice PHP per verificare e validare i dati inseriti dall'utente, salvare il post nel database e reindirizzare l'utente a una pagina specifica, come la pagina del feed o la pagina del post appena creato. Questo file può essere chiamato dall'azione del modulo HTML per inviare i dati e gestire l'interazione con il database.
printLikePhotos.php: Questo file gestisce la visualizzazione dei "Mi piace" ricevuti da una foto. Può includere il codice per recuperare le informazioni sui like dalla foto specifica nel database e formattarle per la visualizzazione nella pagina. Il file può mostrare il numero totale di like e, se appropriato, l'elenco degli utenti che hanno messo "Mi piace" alla foto.
